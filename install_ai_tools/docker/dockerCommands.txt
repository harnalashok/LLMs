## 1. Quick Summary:

	==A Docker image is a standalone, blueprint used to create a container== 
	==A container is an embodiment of what all is soecifed in image==
	
	docker build .	    ==> Builds image from Dockerfile ie from a collection of 
						    instructions as what all the image should contain.
                           dot (.) means Dockerfile exists in the current folder.
	docker run	        ==> Downlods an image from specified docker registry, if specified,
							else, looks for image locally. And then runs the image to 
                            create container.
                            Stop the container with 'docker stop'.
    docker run --rm     ==> Same as above, but will delete the container as soon
                            as it is stopped to save space on hard-disk.
                            Use 'docker run' AND NOT 'docker start' again to 
                            bring container to life but this time, image 
                            is NOT downloaded as it already exists.

	docker compose	   ==> Downloads multiple pre-built multiple images and starts 
                           multiple containers.

	docker start	   ==> Starts a container, only if it exists, but was stopped
                            (not deleted)
	docker stop	       ==>  Stop a running container
 	docker volume	   ==> Docker volumes are meant to provide a persistent storage mechanism.
						    It is used for managing data and storing them outside the container's
						    system ensuring it data persistence even when the container is removed
						    or recreated. These are essential for maintaining data consistency and
						    sharing the data between multiple containers.
						    (Example: docker volume create n8n_data )
						    docker run command can automatically create volumes if a volume is
			 			    specified with the -v
						    The default location of created volumes is /var/lib/docker/volumes/

## 13. Delete docker images
#  	     To remove the image, you have to remove/stop all 
#          the containers which are using it. Here are the steps:
# a. What Images exist?
		docker images
# b. Stop container if running
                docker stop <container_id>: To stop a running container.
		docker stop local-ai
	
# c. Remove/delete a docker container(only if it stopped).
	        docker rm <container_id>
		docker rm local-ai
	
# c.1 Delete all containers:
		docker container rm -f $(docker container ls -aq)
               OR
#            Delete specific multiple containers: 
	       docker container rm webserver laughing_kowalevski relaxed_sammet

####### Useful docker commands: ############333

Common Commands:
  run         Create and run a new container from an image
  exec        Execute a command in a running container (say, ls -la)
	 docker exec -it   92985f90db92  ls -la /

  ps          List containers (Process Status)
  images      List images
  version     Show the Docker version information
  info        Display system-wide information

Management Commands:
  container   Manage containers
  image       Manage images
  volume      Manage volumes

Commands:
  cp          Copy files/folders between a container and the local filesystem
  port        List port mappings or a specific mapping for the container
  restart     Restart one or more containers
  rm          Remove one or more containers
  rmi         Remove one or more images
  start       Start one or more stopped containers
  stats       Display a live stream of container(s) resource usage statistics
  stop        Stop one or more running containers


Global Options:
      --config string      Location of client config files (default "/home/ashok/.docker")
  -v, --version            Print version information and quit

# AA. Image download from docker hub
# Downloads the image, creates container and runs it. 
# Deletes container (--rm) when stopped. Not visible with docker ps -a
# https://github.com/erseco/alpine-php-webserver

 docker run --rm -p 8080:8080 erseco/alpine-php-webserver

# Following will download image, start container but will NOT delete it.
  docker run -p 8080:8080 erseco/alpine-php-webserver
  (Can re-start it as: docker start <name OR id>)

# CC. Build image from Dockerfile
# Ref: https://github.com/mmumshad/simple-webapp-flask

git clone https://github.com/mmumshad/simple-webapp-flask.git
cd simple-webapp-flask
# Build image. Dockerfile in current folder (.)
docker build -t simple-webapp-flask .
# Run image
docker run -p 5000:5000 simple-webapp-flask
(Next time use docker start IF container exists but is topped.
 Else, use 'docker run' again.)
