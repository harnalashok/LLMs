# G. Strategy

What strategy should I adopt and when?  Should I have an Agent with two tasks OR have two Agents with one task each. Can you give examples of which route to take when?

Great, I have solid grounding. Here's the answer:

## Strategy: Specialists Over Generalists

CrewAI's guidance is clear: **prefer multiple specialized agents, each with a focused task, rather than one agent juggling many tasks**. Specialized agents produce more consistent, higher-quality outputs because their role, goal, and backstory align tightly with one objective.

### Use **Two Agents, One Task Each** when:
- The tasks require **different skills / personas** (e.g., gathering data vs. writing prose).
- The tasks need **different tool sets** — exactly your case: Researcher uses `yfinance` + `serpapidevt`; ReportWriter only uses `serpapidevt`.
- You want **clear handoff points** between stages of work.
- You may reuse one of the agents in other crews.

**Example (your scenario):**
- `Researcher` → `research_task` (collect stock data + web context)
- `ReportWriter` → `report_task` (turn research into polished report)

This matches the doc's example crew of Research Specialist → Technical Writer → Editor.

### Use **One Agent, Two Tasks** when:
- Both tasks fall within the **same domain of expertise** and need the **same tools**.
- The second task is a natural continuation/refinement of the first (e.g., "draft outline" then "expand outline" — same writer).
- Splitting would create artificial handoffs with no skill change.

**Example:** A `Technical Blog Writer` agent doing `draft_post_task` then `polish_post_task` — same persona, same tools, sequential refinement.

### Key principles from the docs:
1. **80/20 rule** — spend 80% on task design, 20% on agent design.
2. **Single Purpose, Single Output** — avoid "god tasks" that combine research + analysis + writing.
3. **Complementary skills** — design agents whose specializations complement each other.
4. **Start sequential, not hierarchical** — don't over-engineer.

### Verdict for your setup
Stick with **two agents, one task each**. The tool sets differ (yfinance is Researcher-only), and the work involves two distinct skills (data gathering vs. report writing). That's a textbook case for separation.
