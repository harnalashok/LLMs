What is BrowserStack?
Answer: BrowserStack is a cloud-based platform that lets developers test their websites and mobile apps across many browsers and devices. It helps developers build and test software faster and more accurately. 

Question: What are the features of BrowserStack?
Answer:	Live testing: Test websites and apps on real devices in real time
	Automated testing: Test apps automatically across many browsers and devices
	testing: Test apps manually on real devices
	Test management: Manage tests, including debugging and insights
	Performance testing: Test how fast apps and websites perform
	Visual testing: Test how apps and websites look
	Accessibility testing: Test how accessible apps and websites are
	API testing: Test APIs

Question: What are the Benefits of BrowserStack?
Answer:	Developers can release bug-free software faster
	Developers can test software at scale
	Developers can test apps in real time
	Developers can test apps on a variety of devices

Question: What is the History of BrowserStack? 
Answer:	BrowserStack was founded in 2011 by Ritesh Arora and Nakul Aggarwal. It started as a service to let developers test their websites on Internet Explorer.

Question: What is BrowserStack local used for?
Answer: Local Testing creates a secure, persistent tunnel between your local development/staging environments and the BrowserStack Cloud. With Local Testing, all your web and mobile apps can be tested out of the box, including those hosted on localhost, behind proxies and firewalls and much more.

Question: What is the difference between Selenium and BrowserStack?
Answer: Selenium Grid is excellent for running tests on web applications across multiple browsers simultaneously. BrowserStack offers broader support for both web and mobile apps, allowing testing on real devices in the cloud

Question: What problem does BrowserStack solve?
Answer: BrowserStack's end-to-end unified testing platform is focused on supporting all genres of testing & operations, from functional testing to visual testing, accessibility testing, performance testing, test observability, API testing, and more. Over 50,000 customers rely on BrowserStack to test their web & mobile apps.

Question: Is BrowserStack a startup?
Answer: BrowserStack has over 25,000 paid customers and 2 million registered developers in more than 135 countries. In October 2015, BrowserStack was recognised as a Bootstrap Champ in The Economic Times Startup Awards.

Question: Is BrowserStack an Indian company?
Answer: Notably, the Indian subsidiary BrowserStack Software Private Limited is based out of Mumbai. Founded in 2011 by Arora and Nakul Aggarwal, BrowserStack offers a platform for software testing across various devices and browsers.

Question: How long is a BrowserStack session?
Answer: BrowserStack Automate supports session durations up to 2 hours. If a session is running for more than 7200 seconds (2 hours), the session is stopped, changing the session status to TIMEOUT on the dashboard.

Question: Is BrowserStack safe?

Question: Is BrowserStack cloud-based?
Answer: BrowserStack's cloud-based infrastructure takes all that hassle out of testing. Developers and testers can access 3000+ real browsers and devices at the click of a button, so they can focus on doing what they do best – building awesome websites.

Question: How popular is BrowserStack?

Question: Where is BrowserStack hosted?
Answer: Our application, processing, data collection, and other supporting servers are hosted in the US, Ireland, London, Singapore, Australia, India. The servers are owned by reputable vendors. Each vendor has been thoroughly vetted, and each implement stringent security policies.


Question: Can I use Chrome DevTools to debug websites on mobile browsers? 
Answer: Yes, we have integrated Chrome DevTools on select browsers on real devices (iOS & Android) in Live. To access the same for debugging your website when testing on mobile devices, click on the "DevTools" button on the in-session toolbar. Note, this feature is currently only available only the following mobile browsers on Live : On iOS: Safari AND On Android: Chrome browser
Additionally, you need to be using Live via the Chrome browser to be able to use DevTools with mobile devices. 

Question: Can I test native apps or install apps on BrowserStack’s physical devices? 
Answer: Yes, you can install and test native or hybrid apps on our App Live and App Automate products. App Live allows you perform interactive testing. While App Automate product allows you to run your automated tests on native or hybrid apps using test automation frameworks like Appium, Espresso, XCUITest and EarlGrey. More framework support will come soon. 

Question: What gestures are supported on remote devices? 
Answer: Currently, gestures are supported only on physical mobile devices. Following is the list of supported gestures:

If you are testing from track pad or touch pad:

    Scroll: Two fingers to scroll up/down. You can only scroll a page vertically. Horizontal scrolling is not supported.
    Swipe: Two finger swipe left/right
    Pinch to zoom: Pinch using two fingers to zoom in/out
    Tap: Tap on touch pad to select/click

If you are using a mouse, you can perform the following gestures:

    Scroll: Scroll using the scroll wheel up/down. You can only scroll a page vertically. Horizontal scrolling is not supported.
    Double-click: Hold the left CTRL and then scroll on the mouse wheel to zoom in/out.
    
Question: Can I access the device camera for testing? 
Answer: Yes, you can access the device camera on mobile devices. You can use the Image or Video Injection feature on select devices & browsers to test your website's check scanning, profile image capture, QR/Barcode scanning features, and video-based scenarios.

Question: How do I zoom on a mobile emulator/device? 
Answer: Use your laptop's mouse pad to zoom in and out on a mobile emulator/device. 

Question: Do BrowserStack devices support testing of device vibrations or haptic feedback? 
Answer: BrowserStack devices are hosted on our data centers. Therefore, the functionality to trigger a vibration or haptic feedback is not supported.

Question: What is the default keyboard language and layout on BrowserStack’s desktop and mobile devices? 
Answer: BrowserStack’s desktop and mobile devices use a default US English keyboard layout. If you are a Live customer, you can use a non-default keyboard layout.

Question: Does BrowserStack provide ‘rooted’ or ‘jailbroken’ devices for web or mobile app testing?
Answer: A ‘rooted’ device is an Android device on which a user has root access, which is similar to having administrative access on computers. Similarly, a ‘jailbroken’ device is an iOS device on which restrictions imposed by the iOS operating system have been removed. As a result, you can install applications that Apple does not support.

Question: Can I use the Phone app on BrowserStack devices? 
Answer: The "Phone" app is currently accessible only on devices running on Android versions 10 and above. Note that you are only permitted to access and open the Phone app, but making calls using the app is not allowed for security reasons.

Question:  Why do I see the Restriction Enabled error when accessing default apps on BrowserStack’s mobile devices?
Answer: BrowserStack restricts access to some of the default apps on iOS and Android devices on its public cloud to ensure a secure network of devices to perform testing on. This also ensures all your apps, actions and data are completed cleaned from these devices when your session is over. When you try to access a restricted default app, you will see the “Restriction Enabled” error. However, it is our constant endeavour to allow access to as many of these default apps as possible for you to carry out your tests.

Question: If a new browser version is released, when is it added to BrowserStack? 
Answer: On desktop, we usually add a new browser version within a week of its stable release and also provide access to the beta version of a browser, whenever possible.
For mobile browsers:
    Chrome browsers are upgraded to the latest versions in the First Month of every quarter for Android.
    Firefox browsers are upgraded to the latest versions in the First Month of every half year for Android.
    Samsung browsers are upgraded to the latest versions in the First Month of every half year for Samsung devices.

Question: Is web testing on Windows Server edition the same as Windows Desktop edition? 
Answer: Yes. The license agreements of Windows Desktop editions (XP, 7, 8 and 8.1) do not allow us to provide remote access to our customers. Therefore, we use Windows Server editions with Desktop Experience, and use the versions of Internet Explorer made for Windows Desktop editions on those OSes.
For Windows 10, we use Windows 10 Professional which is a Desktop edition. 

Question: How does BrowserStack decide which browsers to provide for cross-browser testing? 
Answer: We refer to several references of global browser usage statistics and then, based on that information, decide which browser versions to provide for browser testing. Our goal is to have 99.5% of browser coverage. We also retain the older versions already available on the site, till their market share is shown to be either non-existent or negligible. 

Question: Are mobile emulators/simulators good enough to test websites on mobile browsers? 
Answer: It is recommended to test on physical devices instead of simulators for real world test results. The iOS Simulator OR Android emulator is a great tool for rapid prototyping and development of your app. However, it is important to test your app on physical devices as a simulator/emulator and a physical device differ in hardware, API support, framework support & OS features.

Question: Can I test different browsers on mobile devices? 
Answer: Yes. You can.

Question: How does BrowserStack allocate browser and devices for test executions? 
Answer: BrowserStack is an on-demand cloud infrastructure for testing web and mobile apps. We have 19 data centers across 13 locations. When you initiate a test session, BrowserStack allocates a testing platform from the data center nearest to you. These platforms are remote computers or mobile devices with different configurations. If your required testing platform is unavailable (does not exist or is busy), BrowserStack allocates a platform from the next closest data center, and so on and so forth.

Question: 

If the platform is not available in any center, BrowserStack polls them for some time. If no platform is found, the request ends with a No Terminal Available message, which is a rarity and happens in less than 0.0x% of cases.

If you do not specify a platform, an appropriate platform is allocated.

When your session ends, BrowserStack clears your session data from the platform so that your data is not carried over to the next session.
